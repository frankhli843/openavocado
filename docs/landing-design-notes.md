# Open Avocado — landing page design notes

Reference research used to redesign the public landing page (`site/index.html`,
generated by `scripts/site/`). The goal was a page that reads as a modern,
credible technology-learning product, not a bare project README.

> Note: these notes summarize the established, widely-referenced designs of the
> pages below (their landing structures are stable and well documented). They
> are used as *inspiration*, not copied. No copy, assets, or code were taken
> from any of them.

## Pages inspected

1. **Brilliant** — https://brilliant.org
   - Does well: interactive-first hero ("learn by doing"), a short daily-habit
     value proposition, benefit-driven section headers, playful but focused.
   - Borrow: put a real interactive/product visual near the top; lead with the
     outcome, not the feature list.
   - Avoid: heavy cartoon illustration and aggressive subscription upsell.

2. **Khan Academy / Khanmigo** — https://www.khanmigo.ai · https://www.khanacademy.org
   - Does well: mission-first framing ("free, world-class education for anyone,
     anywhere"), audience segmentation (learners / teachers / parents), an AI
     tutor shown as a chat.
   - Borrow: mission line up top, "who it's for" segmentation, show the AI
     tutor as a real conversation.
   - Avoid: institutional density and too many audiences at once.

3. **DeepLearning.AI** — https://www.deeplearning.ai
   - Does well: serious, credible tone; clear learning-path framing; generous
     whitespace; strong typography hierarchy.
   - Borrow: calm credibility, learning-path/"journey" framing, lots of air.
   - Avoid: dumping a full course catalog grid onto the landing page.

4. **Hugging Face (Learn / Hub)** — https://huggingface.co/learn
   - Does well: developer-first and community-first; docs/learn hub as a clean
     card grid; technical but approachable.
   - Borrow: developer + community framing, a docs/runtime card grid, badges to
     signal status.
   - Avoid: crowded logo walls and dense above-the-fold.

5. **Cursor** — https://cursor.com
   - Does well: product-led hero with a real product video/screenshot; tight
     benefit statements; minimal nav; one primary CTA plus a secondary.
   - Borrow: a product video in/near the hero, terse benefit copy, a clear
     primary + secondary CTA pair.
   - Avoid: pure dark theme (Open Avocado is light/green) and download-only focus.

6. **Replit** — https://replit.com
   - Does well: "runs in the browser" emphasis; playful yet technical; clear
     multi-audience paths.
   - Borrow: highlight browser-run code (Open Avocado uses Pyodide), make the
     "run it" paths obvious.
   - Avoid: a busy, over-stuffed hero.

7. **Codecademy** — https://www.codecademy.com
   - Does well: interactive editor preview; skill-path framing; strong
     onboarding.
   - Borrow: show the real in-browser code editor; frame content as paths from
     familiarity to mastery.
   - Avoid: heavy signup gating before any value is shown.

8. **LangChain / LlamaIndex docs** — https://python.langchain.com · https://docs.llamaindex.ai
   - Does well: developer docs hub with a clean card grid; clear information
     architecture; integration/runtime cards.
   - Borrow: docs card grid and runtime/integration cards with honest status.
   - Avoid: doc-only coldness on a page that is also a first impression.

## Patterns adopted for Open Avocado

- **Above the fold:** mission eyebrow ("Empower human learning") + strong H1
  (exact name *Open Avocado*) + one-sentence value proposition + three CTAs
  (*View the demo*, *Read the docs*, *Run locally*) + a direct link to the live
  public site + a real product visual (the showcase video) right under the hero.
- **Show, don't tell:** alternating sections backed by *real* product
  screenshots (dashboard, mastery/adaptive view, interactive visualizations,
  code exercise, quiz, concept map, AI tutor) instead of placeholder cards.
- **How it works:** a compact 4-step adaptive loop (define → evidence → generate
  → familiarity → competence → mastery).
- **Run it your way:** four runtime cards (direct API key, Gemmaclaw, OpenClaw,
  Hermes) with honest READY / PARTIAL / PLANNED badges.
- **Open source & community band** and a **who it's for** audience section.
- **Accessibility:** WCAG-AA contrast, generous spacing, one consistent card
  system, captions/alt text on media, a poster + fallback for the video.

## Patterns deliberately avoided

- Fake testimonials, invented company logos, or unverifiable metrics.
- Dark-pattern signup walls before any value is shown.
- Walls of text, cartoonish over-illustration, or unexplained jargon.
- Any private/internal language (no personal, infrastructure, or task-system
  references anywhere on the public surface).
