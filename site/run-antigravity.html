Quick Start › Run locally

Run with Antigravity prompt

Antigravity is Google's agentic development platform — a separate agent runtime that reads a repository, runs commands, edits files, and executes code. Point it at Open Avocado with the setup prompt below and it will stand the app up and author or review lessons through the documented contract. This is a bring-your-own-agent path: no bespoke Antigravity integration is shipped.

Two different runtimes. Antigravity is the agent runtime that does the work. Open Avocado is the app runtime — a Next.js app plus a pluggable lesson generator. This page tells the agent how to connect the two; it does not claim a turnkey Antigravity plugin.

Who this is for

  • You already use Antigravity and want it to set up, run, and extend Open Avocado for you.
  • You want an agent to author or review lessons against the quality bar instead of wiring generation by hand.

1 · Install & launch Antigravity

Install Antigravity from its documentation — antigravity.google — then launch it in an empty working directory:

# Download the Antigravity editor from https://antigravity.google
# Open an empty folder and start an agent session in it.

Antigravity is a Google agentic development platform in public preview, not an open-source CLI — there is no public source repository, so the link above is the official documentation.

2 · Paste this setup prompt

Copy the prompt below into Antigravity. It uses only documented Open Avocado commands — nothing bespoke:

You are setting up Open Avocado, an open-source adaptive-learning app, from scratch.

1. Clone and read the repository so you understand it:
     git clone https://github.com/frankhli843/openavocado.git
     cd openavocado
   Read README.md and the docs it links (Quick Start, Architecture,
   Configuration, and the Lesson Authoring standard).

2. Install dependencies and create local config:
     pnpm install
     cp .env.example .env.local
   In .env.local set ONE provider key (kept server-side, never committed):
     GOOGLE_AI_STUDIO_API_KEY=<your Google AI Studio key>
     AVOCADOCORE_DEFAULT_PROVIDER=google-ai-studio
     # optional: GOOGLE_AI_STUDIO_MODEL=<model id>
   # or, for OpenAI:  OPENAI_API_KEY=sk-...  and  AVOCADOCORE_DEFAULT_PROVIDER=openai
   Keep in-process generation on:
     AVOCADOCORE_COMPLETION_ADAPTER=local-queue

3. Seed a local database and start the app:
     mkdir -p data
     pnpm db:migrate --seed
     pnpm dev            # http://localhost:3000

4. Verify it is healthy:
     curl -s http://localhost:3000/api/health
   Confirm the default provider is configured and the database answers.

5. Generate and review a lesson:
   Create a subject in the UI (or complete a lesson) so the generator runs.
   Read the generated lesson payload and check it against the lesson-authoring
   quality bar (evidence-first, real interactives, hidden tests, separate QA).
   Improve the config or lesson content until it meets the bar, then write
   validated lessons back through the documented contract (the app API or
   database) — never fabricate content.

6. Prove it works before you finish:
     pnpm test
     pnpm build
   Report the health output, test results, and one generated lesson.

3 · How it connects to lesson generation

Antigravity does not need a special adapter. It drives the same documented contract every runtime uses: run the in-process generator (local-queue) with a configured provider and review its output against the lesson-authoring standard, or act as the external runner behind the dora-task / agent-harness adapters and write validated lessons back. See Configuration for every variable.

Honest status. Open Avocado ships the app, the generator contract, and the adapters — not a packaged Antigravity plugin. Lesson quality depends on the model and on the agent following the quality bar. Treat this as a repeatable setup recipe, not a one-click integration.
